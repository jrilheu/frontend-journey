Clase 2 – Flexbox en profundidad

Duración: 35–45 minutos

Objetivo de esta clase

En la clase 1 viste lo básico del contenedor flex: display: flex, justify-content, align-items y gap. Aquí vamos un paso más allá: aprenderás a controlar cada ítem por separado — cuánto puede crecer, cuánto puede encogerse, qué tamaño base tiene y en qué orden aparece. Con eso podrás montar navbars, filas de tarjetas y centrados que se adapten al espacio disponible.

flex-grow, flex-shrink y flex-basis

Estas tres propiedades css pertenencientes a la familia de propiedades flexbox, se aplican a los ítems flex (los hijos del contenedor), no al contenedor. Deciden cómo se reparte el espacio sobrante o cómo se recorta cuando no hay suficiente.

  • flex-basis: El «tamaño base» del ítem antes de repartir espacio extra. Puede ser un valor como 200px, 10rem o auto (usa el ancho/alto del contenido). Si no lo pones, suele considerarse auto.
  • flex-grow: Si sobra espacio en el eje principal, ¿cuánto de ese sobrante puede «tragar» este ítem? Por defecto es 0 (no crece). Si pones 1 en todos los ítems, se reparten el espacio a partes iguales. Si uno tiene 2 y los demás 1, ese ítem crece el doble que los otros.
  • flex-shrink: Si falta espacio y los ítems tienen que encogerse, ¿cuánto puede encogerse este? Por defecto es 1. Con 0 el ítem no se encoge (puede provocar desborde).

En la práctica, muchas veces quieres «que este ítem ocupe lo que quede»: por ejemplo el contenido principal. Ahí suele bastar flex-grow: 1 (y opcionalmente flex-basis: 0 para que el reparto sea puramente proporcional).

Unidades de medida en CSS

Y ya que hemos visto valores como px, rem o % en los ejemplos, conviene repasar las unidades de medida que se usan en CSS. No hace falta memorizarlas todas; con tener clara la diferencia entre las más habituales te basta.

  • px (píxeles): Unidad fija ligada a la pantalla. Un 16px es siempre el mismo tamaño en píxeles, pero en pantallas de alta densidad puede verse más pequeño. Sigue siendo muy usada para bordes, sombras o cuando quieres un valor exacto.
  • rem: Relativa al tamaño de la fuente del elemento raíz (html). Por defecto en los navegadores, 1rem ≈ 16px. Si el usuario cambia el tamaño de fuente del navegador, todo lo que uses en rem escala con él. Muy recomendable para font-size, márgenes y paddings.
  • em: Relativa al font-size del propio elemento (o del padre). 1.5em = 1,5 veces ese tamaño. Útil para espaciados que quieres que guarden proporción con el texto del bloque.
  • % (porcentaje): Respecto al elemento padre (por ejemplo width: 50% = la mitad del ancho del contenedor). Depende de la propiedad: para anchos y altos suele ser respecto al padre; para fuentes, respecto al font-size del padre.
  • vh y vw: Porcentaje del viewport (la ventana del navegador). 100vh = toda la altura visible; 50vw = la mitad del ancho. Útiles para pantallas completas o secciones a «altura de ventana».
  • fr (fracción): Solo en CSS Grid. Representa una parte del espacio disponible (1fr 1fr = dos columnas iguales). En Flexbox no se usa; ahí el reparto lo hacen flex-grow y flex-basis.

En resumen: para tamaños de texto y espaciado que escale con la accesibilidad, rem; para cosas que dependan del contenedor, %; para «toda la pantalla», vh/vw; y px cuando necesites un valor fijo.

Advertencia: Muchas veces nos veremos tentados a establecer tamaños fijos con px (anchos, altos, fuentes). Hay que tener en cuenta que en móviles las páginas suelen renderizarse peor cuando dependen de tamaños fijos: el contenido puede desbordar, el texto ser ilegible o los botones quedar cortados. Siempre que puedas, prioriza unidades relativas (rem, %, vw/vh) o flexibles (como flex y minmax en Grid) para que el diseño se adapte a distintas pantallas.

La propiedad shorthand: flex

En lugar de escribir flex-grow, flex-shrink y flex-basis por separado, puedes usar flex con uno, dos o tres valores. El orden habitual es:

flex: flex-grow flex-shrink flex-basis;

Algunos ejemplos que verás mucho:

  • flex: 1; — Equivale a flex: 1 1 0%: el ítem puede crecer y encogerse, y su base es 0, así que reparte el espacio disponible por igual con otros ítems que también tengan flex: 1.
  • flex: 0 0 200px; — No crece, no se encoge, y tiene 200px de base. Útil para sidebars o columnas de ancho fijo.
  • flex: auto; — Equivale a flex: 1 1 auto: usa el tamaño del contenido como base y luego puede crecer o encogerse.

order y align-self

Dos propiedades más que se aplican a los ítems y te dan control fino sin tocar el HTML:

  • order: Define el orden visual de los ítems. Por defecto todos tienen 0. Si a un ítem le pones order: -1, aparece antes que los que tienen 0; si le pones order: 1, aparece después. Útil para mover un elemento (por ejemplo un botón «Login») al final de la fila en desktop y al principio en móvil cambiando solo el CSS.
  • align-self: Sobrescribe el align-items del contenedor solo para ese ítem. Valores: auto, start, end, center, stretch, baseline. Por ejemplo puedes tener la mayoría centrados y uno alineado al inicio o al final.

Ejemplo 1: Navbar (menú de navegación)

Una barra con logo a la izquierda y enlaces a la derecha. El contenedor es flex; el logo no crece; los enlaces van en un nav que también es flex y tiene gap.

.header {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 1rem 2rem;
}
.header .logo { flex: 0 0 auto; }
.header nav {
  display: flex;
  gap: 1.5rem;
}

flex: 0 0 auto en el logo hace que no crezca ni se encoja y que ocupe solo el espacio de su contenido. El space-between del contenedor separa logo y nav.

Ejemplo 2: Tarjetas en fila

Varias tarjetas del mismo ancho en una fila que se reparten el espacio. Cada tarjeta puede ser flex: 1 1 0 (o flex: 1) para que todas tengan el mismo ancho y se adapten al contenedor.

.tarjetas {
  display: flex;
  gap: 1rem;
  flex-wrap: wrap;
}
.tarjetas .card {
  flex: 1 1 200px; /* mínimo ~200px, luego se reparten */
}

flex-wrap: wrap hace que si no caben todas en una fila, pasen a la siguiente. Con flex: 1 1 200px cada tarjeta tiene una base de 200px, crece si sobra espacio y se encoge si falta, así que en pantallas anchas verás varias en fila y en pantallas estrechas pueden apilarse.

Ejemplo 3: Centrado perfecto (eje principal y transversal)

Centrar un bloque en medio de la pantalla (o de un contenedor) es muy común. Con Flexbox en el contenedor:

.contenedor {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
}
.contenedor .caja {
  /* sin flex-grow para que la caja no se estire */
}

El contenedor ocupa al menos toda la altura de la ventana (min-height: 100vh). justify-content: center y align-items: center colocan el único hijo (o varios, en bloque) en el centro. Si quieres que la caja interior no crezca, no le pongas flex-grow; con eso se queda a su tamaño natural y centrada.

Resumen: En los ítems flex usa flex-grow, flex-shrink y flex-basis (o el shorthand flex) para repartir espacio; order para cambiar el orden visual; align-self para alinear un ítem distinto al resto. Con eso cubres navbar, tarjetas en fila y centrado sin depender de trucos antiguos.

Comprueba lo aprendido

Practica con un HTML tuyo: crea una barra de navegación con logo y tres enlaces usando Flexbox. Luego haz una fila de tres tarjetas que se repartan el ancho por igual (flex: 1). Por último, centra un div en medio de la página con justify-content y align-items.
