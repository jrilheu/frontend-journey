Clase 4 – Introducción al CSS Grid & Flexbox Mastery

Duración: 35–40 minutos

¿Qué es el CSS y para qué sirve?

CSS (Cascading Style Sheets) es el lenguaje que define la presentación de un documento HTML: colores, tipografías, espaciados, disposición en pantalla y animaciones. El HTML describe la estructura y el significado del contenido; el CSS dice cómo debe verse y, en buena medida, cómo se adapta a distintos dispositivos.

Cómo funciona: El CSS se aplica mediante reglas formadas por un selector (qué elemento(s) afectar) y unas propiedades (qué estilo aplicar). El navegador combina las hojas de estilo, resuelve la cascada y la especificidad, y pinta la página según esas reglas. Puedes escribir CSS en archivos .css enlazados con <link>, en bloques <style> o en atributos style (solo para casos puntuales).

Utilidad en el HTML moderno: En el desarrollo web actual, el CSS es imprescindible para lograr diseños accesibles, responsivos y mantenibles. Permite separar estructura (HTML semántico) de presentación (CSS), reutilizar estilos, trabajar con variables (custom properties), y usar sistemas de layout como Flexbox y Grid, que veremos en esta clase.

En esta primera clase sentamos las bases para dominar dos de las herramientas más potentes del CSS moderno: Flexbox y CSS Grid. Aprenderás qué son, cuándo usar cada uno y los conceptos fundamentales que necesitas para las siguientes clases.

¿Qué son Flexbox y CSS Grid?

Ambos son sistemas de layout que permiten organizar elementos en la página sin depender de floats ni tablas. Resuelven problemas de alineación, distribución del espacio y diseños responsivos de forma declarativa y predecible.

  • Flexbox — Pensado para layouts en una dimensión (filas o columnas). Ideal para barras de navegación, tarjetas en fila, centrado vertical/horizontal y componentes que fluyen en una sola dirección.
  • CSS Grid — Pensado para layouts en dos dimensiones (filas y columnas a la vez). Ideal para cuadrículas, dashboards, galerías y estructuras que necesitan control preciso de filas y columnas.
Regla práctica: Si es una fila o una columna de ítems → Flexbox. Si necesitas filas y columnas definidas (como una tabla o una cuadrícula) → CSS Grid. En muchos diseños usarás ambos juntos.

Bases de Flexbox

Flexbox trabaja con un contenedor flex y sus hijos directos (ítems flex). El contenedor define la dirección y el comportamiento; los ítems se distribuyen según esas reglas.

Conceptos clave

  • Eje principal (main axis): La dirección en la que se disponen los ítems (horizontal por defecto).
  • Eje transversal (cross axis): Perpendicular al principal; sirve para alinear en la otra dirección.
  • flex-direction: row (por defecto), row-reverse, column, column-reverse.
  • justify-content: Alinea los ítems a lo largo del eje principal (start, center, end, space-between, space-around, space-evenly).
  • align-items: Alinea los ítems en el eje transversal (stretch, start, center, end, baseline).

Ejemplo mínimo: contenedor flex

.contenedor {
  display: flex;
  justify-content: center;
  align-items: center;
  gap: 1rem;
}

Con esto, los hijos directos de .contenedor se disponen en fila, centrados en ambas direcciones y con espacio entre ellos (gap).

Bases de CSS Grid

CSS Grid divide el espacio en filas y columnas. Defines la estructura en el contenedor y colocas (o dejas que se auto-coloquen) los hijos en las celdas.

Conceptos clave

  • Grid container: El elemento con display: grid.
  • Grid tracks: Las filas y columnas (definidas con grid-template-rows y grid-template-columns).
  • Celdas (cells): La intersección de una fila y una columna.
  • Grid items: Los hijos directos del contenedor; cada uno ocupa una o más celdas.

Ejemplo mínimo: cuadrícula simple

.cuadricula {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-template-rows: auto auto;
  gap: 1rem;
}

1fr es una unidad de fracción: cada columna ocupa una parte igual del espacio disponible. Los ítems se colocan automáticamente en las celdas de izquierda a derecha y de arriba abajo.

Resumen de esta clase: Flexbox = una dimensión (filas o columnas); Grid = dos dimensiones (filas y columnas). Dominar display: flex, justify-content, align-items y display: grid con grid-template-* y gap es la base para las siguientes clases.

Próximos pasos (Clases 2–5)

  1. Flexbox en profundidad: flex-grow, flex-shrink, flex-basis, orden y alineación por ítem.
  2. Grid en profundidad: líneas, áreas con nombre, grid-column / grid-row, y patrones responsivos.
  3. Patrones reales: cabecera, sidebar, footer y contenido principal con Grid y Flexbox.
  4. Responsive: media queries, auto-fill / auto-fit, y combinación de ambos sistemas.
  5. Proyecto integrador: maquetar una página completa aplicando todo lo aprendido.

Comprueba lo aprendido

¿Cuándo elegirías Flexbox y cuándo CSS Grid para un menú de navegación horizontal? ¿Y para una galería de imágenes en cuadrícula de 3 columnas?

Respuesta breve: menú en una fila → Flexbox. Galería en filas y columnas → CSS Grid.
