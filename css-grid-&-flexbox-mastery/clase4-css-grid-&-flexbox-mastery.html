Clase 4 – Patrones reales y responsive

Duración: 40–50 minutos

Objetivo de esta clase

En las clases anteriores viste Flexbox y Grid por separado. Aquí vamos a juntarlos en un layout real: una página con cabecera, contenido principal, barra lateral y pie de página. Además, aprenderás a hacer ese layout responsive con media queries: cuándo pasar de varias columnas a una sola, cómo adaptar el menú en móvil y qué breakpoints usar. Es el paso previo al proyecto integrador de la clase 5.

Patrón clásico: header, main, aside, footer

El esquema que vas a montar es muy habitual en blogs, dashboards y sitios corporativos:

  • Header: logo a la izquierda y menú en fila a la derecha → Flexbox con justify-content: space-between.
  • Área central: main (contenido) + aside (sidebar) en dos columnas → Grid con grid-template-columns: 1fr 280px o minmax().
  • Footer: varias columnas (enlaces, contacto, redes) → Flexbox o Grid según prefieras.

Todo el layout puede ir dentro de un contenedor con display: grid y grid-template-rows: auto 1fr auto (cabecera, zona central flexible, footer). La zona central es otro grid o flex para main + aside.

.layout {
  display: grid;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}
.header {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 1rem 2rem;
}
.contenido {
  display: grid;
  grid-template-columns: 1fr 280px;
  gap: 1.5rem;
  padding: 1.5rem 2rem;
}
.footer {
  display: flex;
  justify-content: space-around;
  padding: 1rem 2rem;
}

Media queries: el punto de corte

Las media queries permiten aplicar estilos distintos según el ancho de la pantalla (o otras condiciones). La sintaxis básica es:

@media (max-width: 768px) {
  /* estilos que solo aplican cuando el ancho es ≤ 768px */
}
@media (min-width: 769px) {
  /* estilos para pantallas más anchas */
}

max-width significa «cuando la ventana es como máximo X píxeles» (móvil y tablet). min-width significa «cuando la ventana es al menos X píxeles» (desktop). Lo habitual es escribir primero los estilos base (móvil) y luego, con min-width, ir añadiendo el layout de pantallas grandes (enfoque «mobile first»).

Breakpoints típicos

No existe un estándar único; depende del diseño. Valores muy usados (en píxeles):

  • 576px — móviles grandes / phablets
  • 768px — tablets en vertical
  • 992px — tablets en horizontal / pantallas pequeñas
  • 1200px — escritorio

Muchos proyectos usan solo uno o dos breakpoints (por ejemplo 768px y 1024px). Empieza con uno y añade más si el diseño lo pide.

Hacer el layout responsive

En móvil suele interesar que main y sidebar se apilen en una sola columna, y que el sidebar quede debajo del contenido. Con el grid de áreas que vimos en la clase 3, basta con redefinir las áreas y las columnas en la media query:

.contenido {
  display: grid;
  grid-template-columns: 1fr 280px;
  grid-template-areas: "main sidebar";
}
@media (max-width: 768px) {
  .contenido {
    grid-template-columns: 1fr;
    grid-template-areas:
      "main"
      "sidebar";
  }
}

Los ítems con grid-area: main y grid-area: sidebar pasan automáticamente a la nueva disposición. No hace falta tocar el HTML.

Menú adaptable: de fila a columna o hamburguesa

En desktop el menú va en fila (Flexbox). En móvil hay varias opciones:

  • Apilar en columna: Cambiar el contenedor del menú a flex-direction: column y centrarlo. Los enlaces quedan uno debajo de otro.
  • Ocultar y mostrar con botón «hamburguesa»: En móvil el menú está oculto (display: none o fuera de pantalla). Un botón abre/cierra el menú. Eso ya implica un poco de JavaScript para alternar una clase; el CSS solo define cómo se ve abierto y cerrado.

Para empezar, la opción más simple es la columna: sin JavaScript, solo cambiar flex-direction en la media query. Si además quieres que el header no sea una fila sino una pila (logo arriba, menú abajo), puedes hacer el header también en columna:

@media (max-width: 768px) {
  .header {
    flex-direction: column;
    gap: 1rem;
  }
  .header nav {
    flex-direction: column;
  }
}
Consejo: Usa las herramientas de desarrollador del navegador (F12) en modo «responsive» o «toggle device» para ver cómo cambia tu layout al redimensionar. Así compruebas los breakpoints sin tener que cambiar el tamaño de la ventana a mano.

Ejemplo: layout completo con breakpoint

Un esqueleto que une todo lo anterior. Estilos base para desktop; media query para móvil:

/* Base: desktop */
.layout {
  display: grid;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}
.header {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 1rem 2rem;
}
.contenido {
  display: grid;
  grid-template-columns: 1fr 280px;
  grid-template-areas: "main sidebar";
  gap: 1.5rem;
  padding: 1.5rem 2rem;
}
.main   { grid-area: main; }
.sidebar { grid-area: sidebar; }

/* Móvil */
@media (max-width: 768px) {
  .header { flex-direction: column; }
  .contenido {
    grid-template-columns: 1fr;
    grid-template-areas: "main" "sidebar";
  }
}

Comprueba lo aprendido

Monta una página con header (logo + menú en Flexbox), main + sidebar en Grid, y footer. Luego añade una media query para que en pantallas ≤ 768px el sidebar pase debajo del main y el header se convierta en columna. Redimensiona la ventana o usa el modo responsive del navegador para comprobar que los cambios se aplican bien.
