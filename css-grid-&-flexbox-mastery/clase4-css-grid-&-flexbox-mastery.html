Clase 4 – Introducción al CSS Grid & Flexbox Mastery

Duración: 20-25 minutos

¿Qué es el CSS y para qué sirve?

CSS (Cascading Style Sheets) es el lenguaje que define la presentación de un documento HTML: colores, tipografías, espaciados, disposición en pantalla y animaciones. El HTML describe la estructura y el significado del contenido; el CSS, tambien llamado hojas de estilo, dice cómo debe verse y, en buena medida, cómo se adapta a distintos dispositivos.

Cómo funciona: El CSS se aplica mediante reglas formadas por un selector (qué elemento(s) afectar) y unas propiedades (qué estilo aplicar), cada propiedad tiene uno o varios valores a ser aplicados al selector. el formato de una regla es: 

selector { 
        propiedad: valor; 
        propiedad: valor; ... 
    }
El navegador, llamado browser en ingles, como Google Chrome, Firefox, Safari, etc., combina las hojas de estilo, resuelve la cascada y la especificidad, y pinta la página según esas reglas. Puedes escribir CSS en archivos .css enlazados con <link>, en bloques <style> o en atributos style (solo para casos puntuales).

Repasamos los conceptos de resolución de la cascada y especificidad

Cuando varias reglas CSS afectan al mismo elemento, el navegador tiene que decidir cuál aplicar. Para eso usa la cascada y la especificidad. No hace falta memorizar fórmulas: con entender la idea ya evitas muchos dolores de cabeza.

¿Qué es resolver la cascada?

La cascada es el orden en que el navegador aplica las reglas. «Resolver la cascada» significa elegir qué regla gana cuando hay varias para el mismo estilo. Para decidirlo, el navegador tiene en cuenta:

  • Origen: ¿El estilo viene del navegador, de tu hoja CSS o del usuario? Normalmente gana lo que tú defines en tu CSS.
  • Especificidad: Si dos reglas tuyas chocan, gana la que tiene el selector «más fuerte» (ver abajo).
  • Orden en el código: Si todo lo anterior es igual, gana la regla que aparece después en el archivo.

En el día a día suele importar sobre todo la especificidad y el orden: qué selector pesa más y qué declaración escribiste al final.

¿Qué es la especificidad?

La especificidad es el «peso» de un selector. Sirve para saber qué regla gana cuando dos o más selectores aplican al mismo elemento.

De menor a mayor peso (simplificado):

  • Elementos (por ejemplo p, div): poco peso.
  • Clases (por ejemplo .destacado) y pseudo-clases (por ejemplo :hover): más peso.
  • IDs (por ejemplo #cabecera): aún más peso.
  • Estilos en línea (style="..." en el HTML): mucho peso (y suelen dificultar el mantenimiento, por eso se usan poco).

Ejemplo: si tienes .item { color: blue; } y #nav .item { color: red; }, el segundo selector es más específico, así que el texto se verá rojo. Con la práctica irás viendo cuándo un estilo «no aplica» por culpa de la especificidad; entonces podrás simplificar selectores o reordenar reglas.

Utilidad en el HTML moderno: Hoy en día el CSS es lo que hace posible diseños accesibles, que se adapten al móvil o al escritorio, y fáciles de mantener. Te permite dejar el HTML solo para la estructura y el significado, y usar el CSS para todo lo visual: así puedes cambiar el aspecto de la página sin tocar el contenido. Además, con cosas como las variables CSS o los sistemas de layout (Flexbox y Grid), puedes organizar y reutilizar estilos sin repetir código. Eso es justo lo que veremos a continuación.

En esta primera clase vamos a sentar las bases de dos herramientas muy usadas en el CSS actual: Flexbox y CSS Grid. Verás qué son, cuándo conviene usar cada una y los conceptos mínimos que necesitas para seguir con las siguientes clases.

¿Qué son Flexbox y CSS Grid?

Son dos formas de organizar elementos en la página sin depender de trucos antiguos (como los floats o las tablas). Con ellos puedes alinear, repartir el espacio y hacer que el diseño se adapte a distintos tamaños de pantalla de forma clara y predecible.

La diferencia principal es la siguiente:

  • Flexbox trabaja en una sola dirección: una fila o una columna. Sirve muy bien para cosas como un menú de navegación en horizontal, una fila de tarjetas, centrar algo en medio de la pantalla o cualquier bloque donde los ítems van uno detrás de otro en una línea.
  • CSS Grid trabaja en dos direcciones a la vez: filas y columnas. Es ideal cuando necesitas una cuadrícula de verdad: por ejemplo una galería de fotos, un panel de control con varias zonas o cualquier layout donde importe tanto la fila como la columna de cada elemento.
Regla práctica para elegir: Si lo que tienes es una fila o una columna de ítems → Flexbox. Si necesitas filas y columnas definidas (como una tabla o una cuadrícula) → CSS Grid. En muchos diseños acabarás usando los dos: por ejemplo Flexbox dentro de una celda de Grid.

Bases de Flexbox

Para usar Flexbox necesitas un contenedor (el elemento al que le pones display: flex) y sus hijos directos, que pasan a ser «ítems flex». El contenedor es quien decide la dirección y cómo se reparten los ítems; los hijos se adaptan a esas reglas. Solo los hijos directos se comportan como ítems flex; lo que esté dentro de ellos no.

Conceptos que debes conocer

  • Eje principal: Es la línea en la que se colocan los ítems. Por defecto es horizontal (de izquierda a derecha), pero puedes cambiarla.
  • Eje transversal: Es la dirección perpendicular al eje principal. Sirve para alinear los ítems en esa otra dirección (por ejemplo, centrarlos verticalmente cuando el eje principal es horizontal).
  • flex-direction: Define si el eje principal es una fila (row, por defecto) o una columna (column). También puedes usar row-reverse y column-reverse para invertir el orden.
  • justify-content: Alinea los ítems a lo largo del eje principal. Valores típicos: start, center, end, space-between (espacio entre ítems), space-around, space-evenly.
  • align-items: Alinea los ítems en el eje transversal. Por ejemplo center para centrarlos en esa dirección, o stretch para que ocupen todo el alto (o ancho) disponible.

Un ejemplo mínimo: contenedor flex

.contenedor {
  display: flex;
  justify-content: center;
  align-items: center;
  gap: 1rem;
}

Con este código, todos los hijos directos de .contenedor se colocan en fila, quedan centrados en horizontal y en vertical, y tienen un espacio fijo entre ellos gracias a gap. Es un patrón muy común para centrar contenido o alinear botones en una barra.

Bases de CSS Grid

Con CSS Grid divides el espacio en filas y columnas. Tú defines en el contenedor cuántas columnas y filas quieres (y de qué tamaño), y los hijos se van colocando en esas celdas. Puedes dejar que se rellenen solas en orden o indicar en qué celda va cada uno (eso lo verás en clases posteriores).

Conceptos que debes conocer

  • Contenedor grid: El elemento al que le pones display: grid. Solo los hijos directos participan en la cuadrícula.
  • Tracks (pistas): Son las filas y las columnas. Las defines con grid-template-columns (para columnas) y grid-template-rows (para filas).
  • Celdas: Cada «cuadradito» que se forma donde se cruzan una fila y una columna. Un ítem puede ocupar una celda o varias.
  • Grid items: Los hijos directos del contenedor; cada uno es un ítem de la cuadrícula.

Un ejemplo mínimo: cuadrícula simple

.cuadricula {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-template-rows: auto auto;
  gap: 1rem;
}

Aquí estamos diciendo: «tres columnas del mismo ancho y filas que se ajusten al contenido». La unidad 1fr significa «una fracción del espacio disponible»: al poner 1fr 1fr 1fr, las tres columnas se reparten el ancho a partes iguales. Los ítems se colocan solos de izquierda a derecha y de arriba abajo, como si rellenaras una tabla. gap añade espacio entre filas y columnas.

Resumen de esta clase: Flexbox organiza en una dimensión (fila o columna); Grid organiza en dos (filas y columnas). Con que te quedes con display: flex, justify-content, align-items y gap para Flexbox, y con display: grid, grid-template-columns, grid-template-rows y gap para Grid, tienes la base para las siguientes clases.

Comprueba lo aprendido

Piensa en voz alta (o anótalo): ¿usarías Flexbox o CSS Grid para un menú de navegación que va en una sola fila horizontal? ¿Y para una galería de imágenes en cuadrícula de 3 columnas?
