Clase 1 – Introducción a las variables CSS

Duración:15–25 minutos

Objetivo de esta clase

Las variables CSS (en la documentación las verás como custom properties) te permiten guardar un valor —un color, un tamaño, una duración— con un nombre y reutilizarlo en todo tu CSS. Así dejas de repetir el mismo valor por todas partes y, cuando quieras cambiar algo (por ejemplo el color principal de la página), solo tocas un sitio. En esta primera clase verás la sintaxis, dónde declararlas, cómo usarlas y cómo se heredan. No hace falta memorizar todo; con practicar un poco se queda solo.

¿Qué son las variables CSS (custom properties)?

Son propiedades que tú inventas: les pones un nombre que empieza por -- y un valor. El navegador no les da un significado especial (no son como color o margin); solo guardan ese valor. Luego, donde necesites usarlo, escribes var(--nombre) y el navegador lo sustituye. Por ejemplo: si defines --color-principal: #00f5d4; y en varios sitios pones color: var(--color-principal);, cuando cambies --color-principal en un solo lugar, todo lo que use esa variable se actualiza.

Vienen muy bien para temas (colores, fuentes), espaciados que quieras mantener coherentes y cualquier valor que se repita. Esta misma página que estás leyendo usa variables para el fondo, los acentos y el texto: si abres las herramientas de desarrollador y miras el CSS, verás un bloque :root al inicio con varias --nombre. Eso es justo lo que estamos viendo.

Sintaxis: declarar y usar

Para declarar una variable escribes el nombre (siempre con los dos guiones delante) y el valor, igual que cualquier otra propiedad CSS:

--nombre-variable: valor;

El nombre puede llevar guiones, por ejemplo --color-boton. El valor puede ser un color, una longitud, un número, un texto o incluso otra variable (usando var()).

Para usar la variable usas la función var():

propiedad: var(--nombre-variable);

Un ejemplo que puedes copiar y probar:

:root {
  --color-principal: #00f5d4;
  --espaciado: 1rem;
}
.boton {
  background: var(--color-principal);
  padding: var(--espaciado) 1.5rem;
}

Si en ese momento la variable no está definida (porque te equivocaste de nombre o la borraste), la propiedad no se aplica. Más abajo verás que puedes dar un valor de respaldo por si acaso.

Dónde declarar las variables

Puedes declarar variables en cualquier selector. Lo importante es que solo están disponibles en ese elemento y en sus hijos (eso se llama ámbito o scope). Según donde las pongas, cambia quién puede usarlas.

  • :root — Es el elemento raíz de la página (<html>). Las variables que definas aquí están disponibles en toda la página. Es el sitio más habitual para colores, fuentes y todo lo que quieras que sea «global».
  • En un selector de clase o de elemento — Por ejemplo .card { --borde: 2px solid blue; }. Esa variable solo existe dentro de los elementos con clase card y sus hijos. Sirve para que un componente (como una tarjeta) tenga sus propios valores sin repetir el mismo borde en veinte reglas.

Si declaras la misma variable en :root y también dentro de .card, dentro de cada tarjeta gana la versión de .card. Así puedes «sobrescribir» el tema solo en una parte de la página sin tocar el resto.

Valor de respaldo (fallback)

La función var() puede llevar un segundo valor: el que se usará si la variable no existe (o si el navegador no soporta custom properties):

color: var(--color-texto, #333);

Si --color-texto está definida, se usa; si no, se usa #333. El respaldo puede ser otra variable, por ejemplo var(--color-texto, var(--gris-oscuro)). Así te aseguras de que la página siga viéndose bien aunque falte una variable o alguien use un navegador muy antiguo.

Herencia

Las variables CSS se heredan. Si defines --mi-color: red; en body, todos los elementos dentro de body pueden usar var(--mi-color). Si en un hijo (por ejemplo un div con clase .caja) vuelves a declarar --mi-color: blue;, ese div y todo lo que lleve dentro verá blue; el resto de la página sigue con red. Con la práctica verás que eso permite hacer temas por sección o por componente sin duplicar reglas.

Resumen: Declaras con --nombre: valor; en :root (global) o en cualquier selector (solo para ese elemento y sus hijos). Usas con var(--nombre) o var(--nombre, valor-de-respaldo). Se heredan, así que los hijos pueden usar las del padre o redefinirlas. En la clase 2 veremos temas (claro/oscuro), usar variables con calc() y algunas buenas prácticas.

Comprueba lo aprendido

En un archivo CSS tuyo (o en un <style> de prueba), define en :root dos variables: un color, por ejemplo --color-destacado, y un espaciado, por ejemplo --gap. Úsalas en un par de reglas: por ejemplo un .contenedor con gap: var(--gap) y un .destacado con color: var(--color-destacado). Luego cambia el valor de una de las variables y comprueba que los estilos se actualizan solos. Con eso ya has cerrado el círculo de esta clase.
