Clase 2 – Temas, calc() y buenas prácticas

Duración: 30–40 minutos

Objetivo de esta clase

En la clase 1 viste cómo declarar variables, usar var() y entender el ámbito y la herencia. Aquí vamos un paso más: montar un tema (por ejemplo claro y oscuro), combinar variables con calc() para calcular tamaños, y algunas buenas prácticas de nombres y organización. Con esto cierras el módulo de variables CSS con herramientas que verás en proyectos reales.

Temas: claro, oscuro y preferencia del sistema

Un tema es un conjunto de valores (colores de fondo, texto, bordes, etc.) que definen el aspecto de la página. Con variables en :root puedes tener un tema «por defecto» y cambiar solo las variables para pasar a otro tema sin tocar el resto del CSS.

Opción 1: clase en el <html> o <body>

Al usuario le das un botón «Modo oscuro» que añade o quita una clase (por ejemplo theme-dark) en document.documentElement o en body. En el CSS redefines las variables cuando esa clase está presente:

:root {
  --color-fondo: #ffffff;
  --color-texto: #1a1a1a;
}
.theme-dark {
  --color-fondo: #0a0e27;
  --color-texto: #e8f1f5;
}
body {
  background: var(--color-fondo);
  color: var(--color-texto);
}

Tus reglas siguen usando var(--color-fondo) y var(--color-texto); solo cambian los valores de las variables cuando el body (o el html) tiene la clase de tema oscuro. Para alternar la clase con un botón hace falta un poco de JavaScript (o puedes usar solo CSS con la opción siguiente).

Opción 2: preferencia del sistema con prefers-color-scheme

Si quieres respetar el modo claro u oscuro que el usuario ya eligió en el sistema operativo, puedes usar una media query:

@media (prefers-color-scheme: dark) {
  :root {
    --color-fondo: #0a0e27;
    --color-texto: #e8f1f5;
  }
}

Así no hace falta botón: el navegador aplica el bloque cuando el usuario tiene preferencia por oscuro. Puedes combinar ambas cosas: tema por defecto con prefers-color-scheme y un botón que añade theme-dark para sobrescribir manualmente.

Variables y calc()

La función calc() permite hacer operaciones matemáticas en CSS (sumar, restar, multiplicar, dividir). Se combina muy bien con variables: por ejemplo, quieres que un ancho sea «el tamaño base menos un margen».

:root {
  --ancho-max: 900px;
  --margen-lateral: 2rem;
}
.contenido {
  max-width: calc(var(--ancho-max) - 2 * var(--margen-lateral));
}

Importante: al escribir calc(), deja espacios alrededor de los operadores (+, -, *, /) para que el navegador no se confunda. También puedes usar calc() con 100% y var() para layouts responsive, por ejemplo width: calc(100% - var(--sidebar));.

Si en la clase 2 del módulo de flexbox viste unidades como rem, recuerda: puedes guardar un rem en una variable y usarlo dentro de calc() para espaciados coherentes en todo el sitio.

Buenas prácticas (nombres y organización)

  • Nombres que describan el rol, no el valor literal: Mejor --color-texto-principal que --gris, porque si mañana cambias el color a azul, el nombre sigue teniendo sentido.
  • Prefijo común: Muchos proyectos usan --color-, --space-, --font- para agrupar mentalmente y buscar en el editor.
  • Variables «semánticas» de alto nivel: Por ejemplo --color-superficie y --color-texto que apuntan a valores concretos; si cambias el tema, solo cambias esas asignaciones en un sitio.
  • Comentarios en el CSS: Un bloque al inicio del archivo con la lista de variables del tema ayuda a quien llegue después (o a ti dentro de seis meses).

No hace falta aplicar todo desde el primer día: empieza con pocas variables en :root y ve añadiendo cuando veas repetición o cuando quieras cambiar de tema fácilmente.

Resumen: Los temas se suelen cambiar redefiniendo variables en .theme-dark, en @media (prefers-color-scheme: dark) o ambos. Con calc() puedes combinar variables y valores fijos para obtener medidas flexibles. Nombres claros y un bloque de variables documentado hacen el mantenimiento mucho más sencillo.

Comprueba lo aprendido

Prueba A: define en :root --color-fondo y --color-texto para un tema claro; luego añade una media query prefers-color-scheme: dark con otros valores para esas mismas variables. Cambia el modo de tu sistema (o del simulador en DevTools) y comprueba que el fondo y el texto cambian.

Prueba B: define --base: 1rem; y usa padding: calc(var(--base) * 2); en un contenedor. Cambia --base y observa cómo el padding escala.
