Clase 1 – Responsive design mobile-first (Fundamentos)

Duración: 30–40 minutos

Objetivo de esta clase

En estas 4 clases vas a aprender responsive design (mobile-first). En esta primera clase ponemos la base: qué es el responsive design y para qué sirve, qué significa mobile-first, por qué el viewport es clave para que la web se vea bien en móvil, y cómo empezar con layouts flexibles antes de entrar a media queries y breakpoints.

¿Qué es el responsive design?

Responsive design (diseño adaptable o responsivo) es la forma de construir páginas web de modo que el contenido y el layout se adapten al tamaño de la pantalla y al dispositivo: móvil, tablet, portátil, monitor grande, etc. No se trata solo de “que se vea”; se trata de que sea usable: texto legible, botones tocables, menús accesibles y sin tener que hacer zoom horizontal para leer.

En la práctica se consigue combinando HTML semántico, CSS flexible (anchos relativos, Grid, Flexbox, imágenes que escalan) y, cuando hace falta, media queries para cambiar reglas según el ancho o las capacidades del dispositivo.

¿Para qué sirve?

  • Alcanzar a más usuarios: hoy una parte muy grande del tráfico web viene del móvil. Si tu sitio solo está pensado para escritorio, pierdes lectura, ventas o inscripciones.
  • Un solo código base: en lugar de mantener una “versión móvil” y otra “de escritorio” por separado, sueles tener una misma página que se adapta. Eso simplifica mantenimiento y actualizaciones.
  • Mejor experiencia y accesibilidad: tamaños de fuente y espacios que escalan ayudan a quien cambia el tamaño del texto del navegador o usa pantallas pequeñas.
  • Coherencia con buscadores y buenas prácticas: Google y otros buscadores valoran sitios que funcionan bien en móvil; el responsive es el enfoque estándar en desarrollo web moderno.

No hace falta dominar todo de golpe: en esta clase verás los cimientos; en las siguientes profundizarás en breakpoints, componentes, layouts con Grid/Flexbox y detalles como imágenes y pruebas en distintos dispositivos.

Qué significa mobile-first

Mobile-first es una estrategia: primero defines el estilo para pantallas pequeñas (móvil) y después agregas mejoras para pantallas más grandes.

Regla mental: primero funciona en móvil. Luego, con min-width, amplías el layout.

El meta tag viewport

El navegador necesita saber cómo interpretar el ancho del dispositivo. Por eso normalmente se incluye en el <head>:

<meta name="viewport" content="width=device-width, initial-scale=1.0" />

Si falta, muchos móviles “renderizan” como si fuera una web de escritorio, y el diseño se ve extraño aunque tu CSS sea correcto.

Layouts base flexibles (antes de media queries)

Un buen responsive empieza con un layout que ya no se rompe en móvil. Ideas clave:

  • Contenedores centrados: max-width + margin: 0 auto.
  • Anchura relativa: width: 100% o % cuando aplique.
  • Espaciados consistentes: usar rem y/o unidades relativas.

Ejemplo:

.contenedor {
  max-width: 900px;
  margin: 0 auto;
  padding: 1rem;
}

Unidades que ayudan en responsive

Las unidades determinan cómo “escala” tu diseño:

  • %: depende del contenedor.
  • rem: escala con la fuente raíz y suele mejorar accesibilidad.
  • vw/vh: depende del viewport (pantalla visible).
  • clamp(): crea un rango (mínimo / preferido / máximo) para tipografía o espacios.

Media queries en mobile-first

Las media queries cambian estilos según condiciones, normalmente el ancho:

@media (min-width: 768px) {
  /* mejoras para pantallas medianas y desktop */
}

En mobile-first se suele escribir:

  • Primero estilos base (móvil).
  • Luego mejoras con @media (min-width: ...).

Breakpoints: no hace falta usar muchos; empieza con 1 o 2.

Consejo: Abre el navegador, activa el modo responsive y verifica que no aparece scroll horizontal. Si aparece, normalmente es porque algún elemento tiene un ancho fijo o un padding que se sale del contenedor.

Comprueba lo aprendido

Práctica (20 min):

  1. Crea un contenedor con max-width, margin: 0 auto y padding.
  2. Haz que una sección ocupe el ancho disponible (width: 100% o %).
  3. Agrega una media query con min-width: 768px para cambiar un detalle del layout (por ejemplo aumentar el tamaño de un título o pasar a 2 columnas con Grid/Flexbox más adelante).

Revisa en el modo responsive del navegador y confirma que todo sigue legible y sin desbordes.
