Clase 3 – Layouts responsive con Grid y Flexbox

Duración: 35–45 minutos

Objetivo de esta clase

Ya tienes mobile-first, breakpoints y componentes con tipografía fluida. Aquí el foco es organizar bloques en la página con las dos herramientas estándar del CSS moderno: Flexbox y CSS Grid. Verás cuándo conviene cada una, propiedades clave y patrones reales (cabecera, contenido principal, barra lateral, grillas de tarjetas) que escalan de móvil a escritorio.

Flexbox o Grid: regla práctica

Ambos sirven para layout; la diferencia está en el tipo de problema:

  • Flexbox es ideal para una dimensión dominante: filas o columnas de elementos que deben alinearse, repartirse el espacio o envolver (wrap). Piensa en barras de navegación, grupos de botones, filas de iconos o centrar un bloque en el eje que elijas.
  • Grid brilla cuando necesitas una cuadrícula en dos dimensiones: filas y columnas a la vez, áreas con nombre, o columnas que se repiten y se adaptan al ancho (auto-fit / minmax).

En proyectos reales se combinan: por ejemplo un body o .layout en Grid para la estructura global, y Flexbox dentro de la cabecera o de cada tarjeta.

Flexbox para layouts responsive

El contenedor flex define el eje principal con flex-direction (row o column). En móvil suele ir en columna; en un breakpoint pasas a fila.

.barra {
  display: flex;
  flex-direction: column;
  gap: 1rem;
}

@media (min-width: 768px) {
  .barra {
    flex-direction: row;
    align-items: center;
    justify-content: space-between;
  }
}

Propiedades que usarás mucho

  • gap: espacio entre ítems sin márgenes manuales.
  • justify-content y align-items: alineación en el eje principal y cruzado.
  • flex-wrap: wrap: los ítems pasan a la línea siguiente si no caben.
  • flex: 1 en un hijo (atajo de flex-grow): ocupa el espacio libre disponible.

CSS Grid: columnas y áreas

Declaras el contenedor con display: grid y defines pistas con grid-template-columns y, si hace falta, grid-template-rows.

Grilla de tarjetas fluida

Patrón muy habitual: tantas columnas como quepan con un mínimo de ancho:

.lista-tarjetas {
  display: grid;
  gap: 1.25rem;
  grid-template-columns: repeat(auto-fit, minmax(min(100%, 260px), 1fr));
}

min(100%, 260px) evita que en viewports muy estrechos una columna “mínima” fuerce scroll horizontal: en pantallas angostas el mínimo efectivo es el 100% del contenedor.

Layout tipo “app”: cabecera, lateral, contenido, pie

Puedes nombrar zonas con grid-template-areas y reasignarlas en un min-width para pasar de una columna (móvil) a planta con sidebar (escritorio):

.app {
  display: grid;
  gap: 1rem;
  min-height: 100vh;
  grid-template-columns: 1fr;
  grid-template-rows: auto 1fr auto auto;
  grid-template-areas:
    "cabecera"
    "principal"
    "lateral"
    "pie";
}

@media (min-width: 900px) {
  .app {
    grid-template-columns: 220px 1fr;
    grid-template-rows: auto 1fr auto;
    grid-template-areas:
      "cabecera cabecera"
      "lateral principal"
      "pie      pie";
  }
}

.cabecera { grid-area: cabecera; }
.lateral  { grid-area: lateral; }
.principal { grid-area: principal; }
.pie      { grid-area: pie; }

En este ejemplo el lateral va debajo del contenido en móvil; en escritorio salta a la columna izquierda. Si prefieres ocultarlo en pantallas pequeñas, combínalo con los patrones de la Clase 2 (menú hamburguesa, display: none hasta un breakpoint, etc.).

Más patrones útiles

  • Centrado con Flexbox: display: flex; justify-content: center; align-items: center; en un contenedor de altura definida (por ejemplo un hero).
  • Dos columnas desiguales: Grid con 2fr 1fr o minmax(0, 1fr) minmax(0, 320px) para texto + aside; en móvil una sola columna.
  • auto-fit vs auto-fill: con auto-fit las columnas vacías se colapsan y el contenido suele estirarse mejor; auto-fill reserva huecos. Para grillas de productos suele preferirse auto-fit.
Consejo: si algo “no encoge” en Grid o Flexbox, revisa min-width: auto en hijos (por defecto puede impedir el encogimiento). A veces basta con min-width: 0 en el ítem problemático o overflow: hidden en contenedores de texto largo.

Comprueba lo aprendido

Práctica (25–30 min):

  1. Monta una cabecera con logo y enlaces: columna en móvil, fila alineada desde min-width: 768px usando Flexbox.
  2. Crea una grilla de al menos 4 tarjetas con repeat(auto-fit, minmax(...)) y comprueba el comportamiento entre 320px y 1200px.
  3. Implementa un layout de página con Grid y grid-template-areas: al menos cabecera, contenido principal y pie; opcional: barra lateral solo en escritorio.

Usa el modo responsive del navegador y verifica que no haya scroll horizontal ni solapamientos raros al cambiar el zoom al 200%.
