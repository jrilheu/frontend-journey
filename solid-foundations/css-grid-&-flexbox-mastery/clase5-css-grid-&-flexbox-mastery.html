Clase 5 – Proyecto integrador

Duración: 45–60 minutos (o más si te enganchas)

Objetivo de esta clase

Has llegado al cierre del módulo. En las clases 1 a 4 viste las bases de CSS, Flexbox y Grid, las propiedades avanzadas, los patrones con áreas y el diseño responsive. En esta clase vas a aplicar todo junto: maquetar una página completa usando Grid para la estructura general y Flexbox donde encaje (cabecera, menú, footer), con al menos un breakpoint para móvil. No hace falta que sea un sitio real con contenido; puede ser un prototipo con bloques o texto de relleno. Lo importante es que tú decidas el layout, lo montes con CSS y veas cómo Flexbox y Grid trabajan en el mismo diseño.

Estructura de la página que vas a maquetar

Te proponemos el mismo esquema que practicaste en la clase 4, para que lo lleves hasta el final:

  • Cabecera (header): logo o título a la izquierda y menú de navegación en una fila a la derecha. Flexbox con justify-content: space-between y align-items: center.
  • Contenido principal (main) + barra lateral (aside): dos columnas — una más ancha para el contenido y otra para el sidebar. Grid con grid-template-columns: 1fr 280px (o minmax(200px, 280px) para el sidebar). Puedes usar grid-template-areas para dar nombres a las zonas.
  • Pie de página (footer): varias columnas (por ejemplo «Enlaces», «Contacto», «Redes») en una fila. Flexbox con justify-content: space-around o Grid con repeat(3, 1fr).

Todo puede vivir dentro de un contenedor (por ejemplo .layout) con display: grid y grid-template-rows: auto 1fr auto, de modo que la cabecera y el footer ocupen solo su altura y la zona central se expanda. Si prefieres algo más simple al principio, haz solo cabecera + contenido ancho completo + footer; luego añades la barra lateral.

Pasos sugeridos

  1. Escribe el HTML primero. Usa etiquetas semánticas: <header>, <nav>, <main>, <aside>, <footer>. Dentro, pon texto de relleno o divs con clase (por ejemplo .logo, .menu, .content).
  2. Dale al layout principal estructura con Grid. Un contenedor con display: grid, grid-template-rows: auto 1fr auto y min-height: 100vh. Así tienes las tres franjas: cabecera, contenido central, footer.
  3. La zona central (main + aside) puede ser otro grid: grid-template-columns: 1fr 280px y, si quieres, grid-template-areas con nombres como main y sidebar. Asigna cada ítem con grid-area.
  4. Cabecera y footer: display: flex, align-items: center, y justify-content: space-between (o space-around en el footer) para repartir logo, menú y columnas del pie.
  5. Responsive: Con una media query (por ejemplo max-width: 768px), cambia a una sola columna en la zona central (grid-template-columns: 1fr y redefine las áreas para que el sidebar quede debajo del main). Opcional: que el header pase a flex-direction: column en móvil.

Ejemplo mínimo de estructura

Puedes usar este esqueleto como punto de partida y luego personalizarlo:

.layout {
  display: grid;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}
.header {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 1rem 2rem;
}
.contenido-principal {
  display: grid;
  grid-template-columns: 1fr 280px;
  grid-template-areas: "main sidebar";
  gap: 1.5rem;
  padding: 1.5rem 2rem;
}
.main   { grid-area: main; }
.sidebar { grid-area: sidebar; }
.footer {
  display: flex;
  justify-content: space-around;
  padding: 1rem 2rem;
}

@media (max-width: 768px) {
  .header { flex-direction: column; }
  .contenido-principal {
    grid-template-columns: 1fr;
    grid-template-areas: "main" "sidebar";
  }
}

El contenedor .layout define las tres franjas; la cabecera y el footer usan Flexbox; el bloque central usa Grid para main + sidebar. La media query adapta el layout a pantallas estrechas. A partir de aquí puedes añadir colores, tipografía, márgenes y el contenido HTML que quieras.

Consejo: Si algo no cuadra, abre las herramientas de desarrollador del navegador (presionando F12 en tu navegador) y revisa cómo se aplican display: grid y display: flex en cada contenedor. Inspeccionar los ítems y las líneas del grid ayuda mucho a depurar.

Comprueba lo que has hecho

Cuando termines, repasa con esta lista:

  • ¿Usaste Grid para la estructura general (tres franjas) o para la zona main + sidebar?
  • ¿Usaste Flexbox para la cabecera, el menú o el footer?
  • ¿El HTML es semántico (header, main, nav, aside, footer)?
  • ¿Añadiste al menos un breakpoint para que en pantallas pequeñas el layout cambie (por ejemplo sidebar debajo del main)?

No hace falta que quede perfecto a la primera. El objetivo es que experimentes combinando ambos sistemas y que te quedes con una página de referencia que puedas seguir mejorando (más secciones, galería con auto-fit, menú hamburguesa con JavaScript, etc.).
