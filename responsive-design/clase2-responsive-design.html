Clase 2 – Breakpoints, componentes y tipografía responsive

Duración: 30–40 minutos

Objetivo de esta clase

En la Clase 1 viste mobile-first, viewport y tus primeras media queries. Aquí profundizas en dónde cortar el diseño (breakpoints), en cómo pensar componentes que se adapten sin duplicar CSS, y en tipografía fluida para que títulos y texto escalan de forma natural entre móvil y escritorio.

Breakpoints: cuándo y cuántos

Un breakpoint es el ancho (u otra condición) a partir del cual cambias reglas CSS: por ejemplo pasar de una columna a dos, mostrar un menú distinto o aumentar márgenes.

En mobile-first los breakpoints se expresan con min-width: el diseño “base” es móvil y, al superar cierto ancho, añades mejoras.

/* base: móvil */
.tarjeta { padding: 1rem; }

@media (min-width: 768px) {
  .tarjeta { padding: 1.5rem; }
}

¿Qué valores usar?

No existe una tabla mágica universal. Lo habitual es partir de pocos cortes y ajustar según tu contenido:

  • ~480px–600px: teléfonos grandes o móvil en horizontal.
  • ~768px: tablet en vertical; muchos proyectos usan un primer “salto” importante aquí.
  • ~1024px: tablet horizontal / portátiles pequeños.
  • ~1280px o más: escritorio amplio, más columnas o más aire.

Evita definir decenas de breakpoints “por dispositivo”. Mejor: cuando el layout se rompe o el contenido pide más espacio, introduces un nuevo corte. Puedes documentar los valores en variables CSS para mantenerlos coherentes y reutilizarlos en comentarios o en hojas compartidas:

:root {
  --bp-md: 768px;
  --bp-lg: 1024px;
}

@media (min-width: 768px) {
  .layout { gap: 1.5rem; }
}

/* Navegadores recientes: @media (min-width: var(--bp-md)) { ... } */

Componentes responsive

Un componente es un bloque reutilizable (tarjeta, cabecera, lista de productos, pie de página). En responsive conviene que cada componente:

  • Ocupe el ancho disponible del contenedor padre (width: 100%, max-width cuando haga falta).
  • No dependa del ancho de toda la página para “saber” cómo verse: primero que sea flexible por dentro (Flexbox/Grid en el propio bloque).
  • Cambie de forma predecible en breakpoints: misma clase, distintas reglas en media queries (o utilidades si usas un framework).

Ejemplo mental: fila de tarjetas

En móvil: una columna (flex-direction: column o un solo track en Grid). A partir de min-width: 768px: varias columnas con flex-wrap o grid-template-columns: repeat(auto-fit, minmax(240px, 1fr)) para que el número de columnas dependa del espacio.

Regla práctica: si copias y pegas el mismo bloque HTML con clases distintas solo para móvil/desktop, considera un solo componente y media queries (o clases de utilidad) sobre el mismo marcado.

Tipografía responsive

El texto debe seguir siendo legible en cualquier ancho: ni diminuto en móvil ni desproporcionado en pantallas grandes.

Tamaños fluidos con clamp()

clamp(mínimo, preferido, máximo) fija un valor que “vive” entre dos límites. El preferido suele usar vw mezclado con rem para crecer con el viewport sin salirse de rangos razonables:

h1 {
  font-size: clamp(1.5rem, 4vw + 1rem, 2.75rem);
}

p {
  font-size: clamp(1rem, 0.25vw + 0.95rem, 1.125rem);
}

Ajusta los números probando en el modo responsive del navegador hasta que la jerarquía visual (h1 > h2 > cuerpo) se mantenga clara.

Longitud de línea y ritmo

  • Para párrafos largos, un max-width en ch o rem (por ejemplo 45–75 caracteres de ancho) mejora la lectura.
  • line-height un poco mayor en títulos cortos y cómodo (~1.5–1.7) en cuerpo de texto.
  • Revisa contraste y tamaño mínimo de fuente en móvil; el usuario puede aumentar el tamaño base del navegador: usar rem en tipografía ayuda a respetar eso.
Consejo: al elegir breakpoints, redimensiona la ventana lentamente y anota el ancho donde el diseño “pide” un cambio. Esos anchos reales suelen ser mejores que copiar valores de otro proyecto sin contexto.

Comprueba lo aprendido

Práctica (20–25 min):

  1. Define en CSS dos o tres variables --bp-* y úsalas en al menos dos @media (min-width: …).
  2. Crea un componente (por ejemplo una tarjeta con título, texto y botón) que en móvil sea una columna y, desde un breakpoint, muestre imagen y texto en fila o una cuadrícula de varias tarjetas.
  3. Aplica clamp() al font-size de un h1 y de un párrafo; comprueba en 320px y en 1280px de ancho.

Valida que no aparezca scroll horizontal y que los textos sigan legibles al 200% de zoom en el navegador.
